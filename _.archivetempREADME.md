# TaskFlow — MERN Task Management Application

A production-ready, role-based task management platform built with the MERN stack. Featuring JWT authentication, email OTP verification, Cloudinary image uploads, Chart.js dashboards, and CI/CD via GitHub Actions.

## 🚀 Quick Start (Local Development)

### Prerequisites
- Node.js >= 18
- MongoDB Atlas account (or local MongoDB)
- Cloudinary account
- Gmail account (for SMTP, or any SMTP provider)

### 1. Clone the repository

```bash
git clone https://github.com/your-username/taskflow.git
cd taskflow
```

### 2. Configure environment variables

**Backend:**
```bash
cd server
cp .env.example .env
```

Edit `server/.env`:
```env
PORT=5000
NODE_ENV=development
MONGO_URI=mongodb+srv://<user>:<pass>@cluster.mongodb.net/taskflow
JWT_SECRET=your_32_char_minimum_secret_here
JWT_EXPIRES_IN=7d
ADMIN_EMAIL=admin@yourcompany.com
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your@gmail.com
EMAIL_PASS=your_gmail_app_password
EMAIL_FROM="TaskFlow <noreply@taskflow.app>"
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret
CLIENT_URL=http://localhost:5173
```

> **Gmail App Password:** Go to Google Account → Security → 2-Step Verification → App Passwords. Generate one for "Mail".

**Frontend:**
```bash
cd ../client
cp .env.example .env
```

Edit `client/.env`:
```env
VITE_API_BASE_URL=http://localhost:5000/api/v1
```

### 3. Install dependencies

```bash
# From project root
cd server && npm install
cd ../client && npm install
```

### 4. Run both servers

**Terminal 1 — Backend:**
```bash
cd server
npm run dev
# → http://localhost:5000
```

**Terminal 2 — Frontend:**
```bash
cd client
npm run dev
# → http://localhost:5173
```

### 5. Create your admin account

Register with the email matching `ADMIN_EMAIL` in your `.env`. The system auto-assigns the `admin` role to that email.

## 🔐 Authentication Flow

```
POST /api/v1/auth/signup    → Creates user, sends 6-digit OTP via email
POST /api/v1/auth/verify    → Verifies OTP (expires in 10 min), returns JWT
POST /api/v1/auth/login     → Returns JWT on valid credentials
GET  /api/v1/auth/me        → Returns current user (requires JWT)
POST /api/v1/auth/logout    → Client-side token clear
```

## 📡 API Reference

### Base URL
```
/api/v1
```

### Headers (protected routes)
```
Authorization: Bearer <jwt_token>
```

### Task Endpoints
| Method | Endpoint | Role | Description |
|--------|----------|------|-------------|
| GET | `/tasks` | Admin | All tasks (filterable by status/priority/search) |
| GET | `/tasks/my` | User | Tasks assigned to current user |
| GET | `/tasks/stats` | Admin | Aggregated stats for dashboard |
| GET | `/tasks/:id` | Both | Single task |
| POST | `/tasks` | Admin | Create task |
| PUT | `/tasks/:id` | Admin | Update task |
| DELETE | `/tasks/:id` | Admin | Delete task |
| PATCH | `/tasks/:id/checklist` | Both | Toggle checklist item |

### User Endpoints
| Method | Endpoint | Role | Description |
|--------|----------|------|-------------|
| GET | `/users` | Admin | All team members |
| GET | `/users/:id` | Admin | User + their tasks |
| PUT | `/users/profile` | User | Update name / profile image |
| DELETE | `/users/:id` | Admin | Remove member |

## 🌐 Deployment

### Backend → Render

1. Push your code to GitHub
2. Go to [render.com](https://render.com) → **New Web Service**
3. Connect your GitHub repo, set **Root Directory** to `server`
4. Configure:
   - **Environment:** Docker
   - **Dockerfile Path:** `./Dockerfile`
   - **Port:** `5000`
5. Add all environment variables from `.env.example` under **Environment**
6. Set `CLIENT_URL` to your Vercel frontend URL
7. Deploy — Render auto-redeploys on `main` pushes

### Frontend → Vercel

1. Go to [vercel.com](https://vercel.com) → **New Project**
2. Import your GitHub repo, set **Root Directory** to `client`
3. Framework: **Vite**
4. Add environment variable:
   - `VITE_API_BASE_URL` = `https://your-render-app.onrender.com/api/v1`
5. Deploy — Vercel auto-redeploys on `main` pushes

### CI/CD (GitHub Actions)

Add these **GitHub Secrets** (Settings → Secrets → Actions):

| Secret | Value |
|--------|-------|
| `RENDER_DEPLOY_HOOK_URL` | From Render dashboard → Deploy Hooks |
| `API_HEALTH_URL` | Your Render service URL |
| `VITE_API_BASE_URL` | Your Render API URL |

The pipeline runs on every PR: lint → tests → build. On merge to `main`, it triggers a Render deploy and health-checks the `/api/v1/health` endpoint.

## 🔒 Security Features

| Feature | Implementation |
|---------|----------------|
| Password hashing | bcryptjs, 12 salt rounds |
| JWT signing | HS256, configurable expiry |
| Auth rate limiting | 10 requests / 15 min per IP |
| Global rate limiting | 200 requests / 15 min per IP |
| NoSQL injection prevention | express-mongo-sanitize |
| HTTP security headers | Helmet.js |
| CORS | Restricted to CLIENT_URL origin |
| Input validation | express-validator on all routes |
| File upload safety | mimetype check + 5MB limit |
| Role guards | Middleware on every protected route |
